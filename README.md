# Система автоматической классификации нормативно-правовых актов

Данный проект представляет собой изолированную программную систему для мультиклассовой классификации муниципальных и федеральных правовых документов. Модель выполняет распределение актов по первому уровню иерархии кодов нормативных классификаторов (19 макро-категорий).

## Технологический стек
* Язык программирования и среда разработки: Python, Jupyter Lab.
* Библиотеки машинного обучения и моделирования: Scikit-Learn, LightGBM (с поддержкой GPU).
* Инструменты обработки естественного языка (NLP): Pymorphy3, Регулярные выражения (Regex).

## Реализованные инженерные решения
1. Изоляция от утечек данных (Data Leakage): Разработан многоэтапный конвейер регулярных выражений, полностью ликвидировавший хронологический и технический OCR-шум (включая разорванные случайными пробелами даты, регистрационные номера документов и цепочки технических нулей).
2. Сохранение юридической семантики: Из списка стоп-слов исключена отрицательная частица "не", критически влияющая на правовой смысл норм. Шаблон токенизации TF-IDF расширен до n-грамм для захвата устойчивых биграмм.
3. Оптимизация предобработки: Внедрен алгоритм лемматизации со строгим кэшированием уникальных токенов через декоратор lru_cache, что сократило время обработки 50 000 текстовых строк до минимума в рамках одного потока.
4. Борьба с дисбалансом классов: Применена математическая стратегия выравнивания весов class_weight='balanced'. Это позволило полностью устранить деградацию метрик на дефицитных классах и подняло показатель полноты (recall) миноритарных категорий на слепом тесте до 1.00.

## Воспроизводимый запуск через Docker
Проект полностью контейнеризирован. Развертывание среды и запуск вычислений происходят в изолированном контуре, не требуя ручной настройки CUDA-драйверов и локального окружения Python.

1. Сборка Docker-образа:
   ```bash
   docker build -t legal_nlp_project .
   ```

2. Запуск контейнера:
   ```bash
   docker run -p 8889:8889 legal_nlp_project
   ```

3. Работа с интерактивной средой:
   Для входа в интерфейс Jupyter Lab перейдите в браузере по адресу http://localhost:8889. Все необходимые предобученные веса моделей и векторизатора автоматически подгрузятся из директории models.
